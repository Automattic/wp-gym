# Diagnose admin/editor browser performance from runtime metrics

Use browser evidence from the WordPress runtime to inspect the admin/editor surface.

1. Navigate to `/wp-admin/` or an editor/admin page available in the runtime.
2. Capture browser evidence with runtime metrics.
3. Report the metric values you observed, especially resource count, transfer size, DOM node count, iframe count, heap usage, and long-task totals.
4. Explain whether the observed page looks lightweight or needs further investigation.

Keep the answer evidence-backed. Do not estimate metrics from elapsed command duration.
